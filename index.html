Abstract

Current cross-view self-supervised methods for 3D vision rely on pixel reconstruction, which may overemphasize low-level appearance over the geometric correspondences needed for downstream tasks like relative pose and optical flow estimation. We propose GATOR, a two-view jigsaw pretraining objective in which a Vision Transformer predicts the original positions of shuffled patches from one view by cross-attending to a clean reference view, recasting pretraining as discrete patch-position classification rather than reconstruction. Compared against CroCo, single-view jigsaw, and MAE baselines under matched compute, GATOR representations transfer better to downstream tasks, showing that geometry-centric objectives yield stronger correspondence features than pixel-level ones.

I. Introduction

Self-supervised learning has become a standard paradigm for learning visual representations from unlabeled data. Methods such as DINO [3] and MAE [4] achieve strong results through self-distillation and masked image reconstruction, respectively. Masked modeling has further been extended to two-view pretraining with CroCo [2], which shows strong performance on 3D vision tasks. Another successful self-supervised paradigm is jigsaw puzzle solving, but its use for 3D computer vision remains largely unexplored.

In this work, we investigate whether a two-view jigsaw objective can serve as an effective pretraining strategy for 3D tasks such as image matching and optical flow. Given two images of the same scene, we keep one view unchanged and shuffle the patches of the other. The model is trained to recover the correct patch arrangement by exploiting the reference view, thereby encouraging the learning of spatial structure and cross-view correspondences. Our approach is not formlauted as pixel-level prediction: instead, it formulates the pretraining objective as a patch reordering problem. This objective still requires 3D understanding to be solved, but without wasting parameters for low-level detail reconstruction such as textures.

III. Methodology

III-A. Architecture

Gator is an encoder-decoder architecture. The encoder is equivalent to ViT-Small: it consists of 12 layers, a hidden dimension of 384 and 6 attention heads. The decoder is 6 layer transformer with same hidden dimension and attention heads as the encoder. Following [1] we add four learnable register tokens.
Two different views of the same scene are passed to the model — one with shuffled patches (of size 16x16) called shuffled and the other one is called reference. The model predicts how to reorder the shuffled view to match the reference one.
We firstly patchify both views and linearly project them to the hidden dimension. Then, we concatenate the register tokens with the patchs tokens and process views using the same encoder. We turn off RoPE in the encoder of the shuffled view to prevent the model from predicting position information based on RoPE coefficients.
Then the decoder processes the shuffled view tokens as queries and the reference view tokens as context. We add sinusoidal positional embeddings to the reference view tokens instead of RoPE, since its coefficients for tokens at the same positions in two views might correspond to different 3D locations.
Finally, the output head processes each token independently and predicts either a position (logit or 2d coordinate) or opacity, depending on the loss used. We experiment with four different losses, which are described below.

III-B. Jigsaw Objectives


Let the input image has size $N \times N \times C$ and patch size $p$, yielding an $n \times n$ patch grid with $n = N/p$. In the description below we interchange 2D in 2D grid and 1D positions a flattened 2D grid for simplicity.
Classification Loss A linear head maps each token to a logit $\hat{y}_k = W h_k + b, \hat{y}_k \in \mathbb{R}^{n^2}$. For token $k$, the target $y_k$ is the index of its true position in the grid. We train with cross-entropy against the true position. To recover predictions we simply use $\arg\max_j \hat{y}_{k,j}$. \[ \mathcal{L_1} = \operatorname{CrossEntropy}(\hat{y}_k, y_{k}) \] Distance-Based Loss A linear head maps each token to a normalized 2D coordinate: $\hat{u}_k = \tanh(W h_k + b), \hat{u}_k \in [-1,1]^2$. The target coordinate $u_k$ is the normalized true position of the patch. We train with MSE agains the true position. To recover the discrete position we map predictions from $[-1,1]$ to $[0,n-1]$ and round to the closest integer. \[ \mathcal{L_2} = \operatorname{MSE}(\hat{u}_k, u_{k}) \] Visual Loss A linear head maps each token to $n^2$ values, followed by $\texttt{softmax}$: $o_k = \mathrm{softmax}(W h_k + b), o_k \in \mathbb{R}^{n^2}.$ Here, $o_{k,i}$ is the predicted opacity for placing patch $k$ at position $i$. Let $z_k \in \mathbb{R}^{p^2C}$ be the flattened ground-truth patch corresponding to token $k$. We softly place each patch at all positions and sum over patches: \[ \hat{Z}_i = \sum_{k=1}^{n^2} o_{k,i} z_k, \qquad \hat{Z} = \{Z_1, \dots, Z_{n^2}\}, \qquad \hat{Z} \in \mathbb{R}^{n^2 \times p^2C}. \] The loss is MSE against the ground-truth patchified image $Z$. This objective naturally handles cases where visually identical patches are swapped, since such swaps do not increase the reconstruction error. \[ \mathcal{L_3} = \operatorname{MSE}(\hat{Z}, Z) \] Similarity Loss This loss is inspired by both cross-entropy loss and visual loss. We use classification, but do not penalize model if it predicts swapped position for visually similar patches. A linear head maps each token to a logit $\hat{y}_k = W h_k + b, \hat{y}_k \in \mathbb{R}^{n^2}$. For token $k$ a prediction distribution is calculated as $p_{k}^j = \texttt{softmax}(-\frac{||\texttt{z}_k - \texttt{z}_j||_2^2}{\tau p^2 C})$, where $\tau$ is a hyperparameter, and $p^2 C$ is a normalization factor corresponding to the patch dimension. The loss is a cross-entropy against the target distribution. \[ \mathcal{L_4} = \operatorname{CrossEntropy}(\texttt{y}_{k}, \texttt{p}_{k}) \]

III-C. Pre-training Details

Taking into account the similarity of the transformer, we actually do not shuffle patches of the shuffled view during training. Since we do not provide any kind of positional information to the model for this view, this does not simplify the task.
Our experiments with Jigsaw model suggest that it is possible to achieve good performance without relying on the reference view. However, for efficient 3D performance the reference view is crucial, therefore our model has to learn to utilize it. To encourage the model to do so, we randomly mask from 35% to 85% of the shuffled patches during training. This way, the model cannot rely on the shuffled view alone and has to learn to use the reference view to solve the task. In the appendix we show that this strategy is effective in encouraging the model to utilize the reference view.

IV. Results

IV-A. Optimization Details

We render approximately 1.8M image pairs following CroCo [2]. We shard dataset into 181 shards and randomly split it into 4 validation and 177 training shards. Sharding enhances GPU utilization from 60% to 90%.
We train our model and competitors on this dataset with compute budget of 24 hours on a single V100 GPU. We tune learning rates of CroCo, Jigsaw and Gator on the validation set based on the pretraining task performance. We use learning rate of $1.5 \cdot 10^{-4}$ for CroCo, $10^{-4}$ for Gator-Classification and Jigsaw, and $3 \cdot 10^{-4}$ for Gator-Distance-Based. We use $\texttt{AdamW}$ with weight decay of $5\cdot 10^{-2}$ for CroCo and $10^{-2}$ for other models. We use a batch size of 128 and a patch size of 16 for all models. We use a cosine learning rate scheduler with 1 warmup epoch. We use FlashAttention and train in $\texttt{fp16}$ precision.

IV-F. Second View Utilization: Zero Reference

In this section we test whether the model utilizes the second view to solve the task, or relies purely on the shuffled view.

IV-G. Second View Utilization: Tiny Dataset

We collect a tiny dataset of 15 images. For each image $x_1$ there is a feature that we can swap some of its patches and the resulting image $x_2$ still looks realistic and $x_1 \neq x_2$. We shuffle patches of $x_1$ to get a shuffled view $x_s$ and then we pass images to the model. If we get the results \[ \operatorname{Gator}(x_s, x_1) \approx x_1, \qquad \operatorname{Gator}(x_s, x_2) \approx x_2 \] it means that the model is able to utilize the second view and solve the task correctly.

Top row is shuffled image before shuffling, middle row is the reference view, and the bottom row is the shuffled view after shuffling. The picture is a carousel. Tap to activate scrolling.
One can observe that the model correctly utilizes the second view and solves the task for all 30 (15 dataset images give 30 reference views) images. Moreover, the prediction accuracy is more than 98% for swapped patches. This suggests that the model learns to utilize the second view to solve the task, rather than relying on shortcuts in the shuffled view.
